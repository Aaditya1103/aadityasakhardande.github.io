Time-Based Grid Search & Multi-Agent Path Planning

GitHub

A technical deep dive into Space-Time A*, Safe Interval Path Planning (SIPP), and Priority-Based Multi-Agent Planning. Focus: state-space formulations, optimality constraints, temporal safety invariants, and complexity properties.

Time-augmented search expands the discrete grid into a 3D lattice ℤ² × ℤ, enabling planners to reason over spatiotemporal feasibility. Moves become transitions in a time-indexed transition system, providing collision-avoidance guarantees against dynamic obstacles and other agents.

Abstract

This document provides a technical overview of time-parameterized grid search algorithms used for dynamic obstacle avoidance and multi-agent path coordination. We formalize the state lattice, define safe intervals, derive the successor-time feasibility constraints, compare asymptotic expansion bounds, and summarize how priority-based planning integrates these primitives via reservation tables. Reduced but illustrative pseudocode is included for critical operations such as time-feasible successor generation and safe-interval merging.

Introduction

Given a grid G = (V,E) and a set of spatiotemporal obstacle constraints O(x,y,t), the feasibility problem becomes finding a sequence:

π = { (x₀,y₀,t₀), (x₁,y₁,t₁), ..., (xₖ,yₖ,tₖ) }

such that:

  • (xᵢ,yᵢ) ∈ V and (xᵢ,yᵢ,tᵢ) ∉ O
  • (xᵢ,yᵢ) and (xᵢ₊₁,yᵢ₊₁) are adjacent (or identical if waiting)
  • tᵢ₊₁ = tᵢ + Δt, commonly Δt = 1
  • No edge conflicts: ((xᵢ,yᵢ) → (xᵢ₊₁,yᵢ₊₁)) ≠ ((xⱼ₊₁,yⱼ₊₁) → (xⱼ,yⱼ)) for any agent pair

Planner Architecture

  • State Representation: s = (x, y, t)
  • Transition Model: deterministic, 4-/8-connected + wait action
  • Time Feasibility: successor states must satisfy free-space and free-edge constraints
  • Single-Agent Planner: Space-Time A* or SIPP
  • Multi-Agent Coordinator: Priority-based planning with a reservation table R(t)

Methodology

1. Space-Time A* (STA*)

STA* applies A* to the time-augmented lattice. The cost metric is temporal: g(s) = t. Admissible heuristics use minimum-time lower bounds:

h(s) = L₁(s, goal) / vmax

Key property: If h is time-admissible and no negative delays exist, STA* is optimal in time relative to the discretization.

STA* Successor Generation 

                            def sta_successors(s):
                                x,y,t = s
                                for nx,ny in neighbors(x,y):
                                    nt = t + 1
                                    if not blocked(nx,ny,nt):
                                        yield (nx,ny,nt)

This expansion quickly becomes large because |V×T| grows linearly with horizon length T. This motivates the SIPP reduction.

2. Safe Interval Path Planning (SIPP)

For each cell c = (x,y), precompute maximal time intervals during which the cell is provably unoccupied:

I(c) = { [t₀,t₁], [t₂,t₃], ... }

A SIPP node is (x, y, I = [t_low, t_high], arrival_t). This drastically reduces branching, because SIPP searches over intervals, not individual time steps.

Safe Interval Extraction (conceptual)

                            def extract_intervals(occupancy_vector):
                                intervals = []
                                t = 0
                                while t < Tmax:
                                    if occupancy_vector[t] == FREE:
                                        start = t
                                        while t < Tmax and occupancy_vector[t] == FREE:
                                            t += 1
                                        intervals.append((start, t-1))
                                    t += 1
                                return intervals

3. Priority-Based Multi-Agent Planning

Agents are sorted by priority π₁ ≻ π₂ ≻ … ≻ πₙ. For each agent k:

  1. Use STA* or SIPP with a modified blocked(x,y,t) that checks the reservation table R
  2. Insert the resulting trajectory into R
  3. Ensure no vertex conflicts: (x,y,t) occupied by multiple agents
  4. Ensure no edge conflicts: simultaneous swaps

The reservation table is typically:


                            R = { t: set((x,y)), 
                                ("edge", t, (x1,y1),(x2,y2)): True }

Priority planners are NOT optimal but extremely efficient in moderately coupled domains.

Technical Analysis

State-Space Complexity

Let |G| be number of grid cells, T the time horizon.

  • STA*: O(|G|·T) nodes
  • SIPP: O(|G|·|I(c)|) where |I(c)| ≪ T; often reduces expansions by ≥ 10×
  • Priority Planner: O(n · C_single) where n agents and C_single is single-agent cost

Collision Constraints

A multi-agent plan must satisfy both:

  • Vertex Safety: ∀ agents i≠j, (xᵢ(t),yᵢ(t)) ≠ (xⱼ(t),yⱼ(t))
  • Edge Safety: (xᵢ(t),yᵢ(t)) → (xᵢ(t+1),yᵢ(t+1)) ≠ (xⱼ(t+1),yⱼ(t+1)) → (xⱼ(t),yⱼ(t))

Heuristic Construction

Time-admissible heuristic:

h(s) = ⌈ ||pos(s) − goal||₁ / vmax ⌉

Conclusion

Time-based planning elevates grid navigation to a full spatiotemporal scheduling problem. STA* provides a baseline with strong optimality guarantees but suffers from time-explosion. SIPP reduces dimensionality by representing maximal contiguous free-time sets as safe intervals, yielding one of the most efficient exact planners for dynamic environments. Priority-based planning overlays these single-agent planners with an efficient collision-avoidance mechanism suitable for multi-robot systems with moderate coupling.

Challenges

  • Large time horizons ⇒ exponential search growth
  • Narrow passages ⇒ priority planners frequently fail without backtracking
  • Dynamic obstacle uncertainty ⇒ STA*/SIPP require deterministic predictions
